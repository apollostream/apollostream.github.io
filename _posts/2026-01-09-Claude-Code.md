I've tried a number of AI-assisted coding environments (IDE's) and strategies: from Cursor, to Manus, to Perplexity.AI "Labs" mode, to Google AI Studio "Build" mode, etc.  All have impressed me up to a point. But all have failed me once I got past that initial phase of creating the UI for an app and got down to digging into making it work.  These AI coding systems tend to eat their own tail as they revise the revised and revise the stuff that you're satisfied with and don't want untouched. And they never really get down to being able to accurately code up the core backend capability at the heart of an interesting app.

That is, until now with the latest release of Anthropic's Claude Code.  Though not perfect, this multi-agent AI coding system is the best I've used. I truly think it signals a step-change in automatic coding capability that we've just not had before.

By now, there are a bunch of raves on the internet. And, I wanted to see what a BFIH analysis might turn up in terms of more substantial assessment of Claude Code relative to the current competition.

And so, here's the first BFIH analysis I'm posting about that was generated by the "BFIH Tournament" app built by me "talking" to Claude Code. It's the first app I've built where I didn't alter any of the code manually.  However, I still used Perplexity.AI to write the blog post below after feeding it the analysis report from the web-app written by Claude Code.
